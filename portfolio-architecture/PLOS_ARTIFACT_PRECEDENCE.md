# PLOS Artifact Precedence

This document defines the order of authority for architectural interpretation within the Product Leadership Operating System (PLOS).

## Precedence Order

1. Portfolio control artifacts in `portfolio-architecture/`
2. Canonical pillar system definitions
3. Pillar supporting models
4. Playbooks and execution guides
5. Diagrams and rendered visuals
6. README summaries and explanatory text

## Rules

- If two artifacts conflict, the higher-precedence artifact governs.
- Diagrams must reflect canonical architecture and may not create new system behavior.
- Playbooks may define execution patterns only and may not redefine ownership boundaries.
- Labs artifacts may not become canonical without explicit governance approval.
