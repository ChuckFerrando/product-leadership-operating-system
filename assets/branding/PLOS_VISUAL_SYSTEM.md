# PLOS Visual System

## Purpose

This document defines the canonical visual rules for all rendered diagrams in the Product Leadership Operating System (PLOS).

## Rules

- Use one consistent visual system across all pillar diagrams
- Preserve pillar naming exactly
- Preserve the canonical loop exactly:
  Strategy → Governance → Delivery → Outcomes → Learning → Strategy
- Do not depict Decision Intelligence as directly informing Strategy or Governance
- Enforce the mediation path visually:
  Decision Intelligence → Customer Outcomes → Strategy / Governance
- Separate Playbooks from Labs visually and conceptually
- Portfolio-level diagrams belong in `assets/diagrams/`
- Pillar-local diagrams must not conflict with portfolio-level diagrams
