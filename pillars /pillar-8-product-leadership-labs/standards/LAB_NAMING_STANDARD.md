# Lab Naming Standard

This document defines the canonical naming convention for experiment artifacts within **Product Leadership Labs**.

---

# Core Principle

> **Experimental artifacts must be clearly identifiable as experimental**

Naming must make it obvious that an artifact belongs to Labs and is not canonical by default.

---

# Format

Use uppercase, underscore-separated, scenario-based names.

## Experiment Artifacts

```text
[SCENARIO]_EXPERIMENT.md
```

# Example Names

- `PRIORITIZATION_SIGNALING_EXPERIMENT.md`
- `RELEASE_COORDINATION_EXPERIMENT.md`
- `OUTCOME_REVIEW_VARIANT_EXPERIMENT.md`

---

# Rules

All Lab artifact names must:

- use uppercase  
- use underscores  
- be scenario-based  
- end with the correct suffix  
- avoid vague or generic labels  

---

# Anti-Patterns

Avoid names such as:

- `test.md`
- `idea.md`
- `model.md`
- `framework.md`
- `pilot.md`

These names are too vague and do not clearly indicate experimental status.

---

# Summary

The **Lab Naming Standard** ensures that Lab artifacts are:

- recognizable  
- consistent  
- clearly experimental  
- easy to review and govern
