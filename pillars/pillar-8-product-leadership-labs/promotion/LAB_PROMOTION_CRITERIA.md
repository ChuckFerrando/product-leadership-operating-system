# Lab Promotion Criteria

This document defines the canonical criteria an experimental artifact must satisfy before it can be promoted from **Product Leadership Labs** into a canonical pillar of PLOS.

---

# Core Principle

> **Promotion requires validation, boundary integrity, and governance approval**

Promising experimentation alone is not sufficient.

---

# Required Criteria

An artifact is promotion-eligible only if all of the following are true:

## 1. Structured Experimentation
- the artifact was created through a bounded experiment  
- the experiment followed the canonical template  
- observations are documented and reviewable  

## 2. Outcome Evaluation
- experiment results were evaluated through the **Customer Outcomes System**  
- meaning and value were not inferred directly from raw signals  

## 3. Boundary Integrity
- the artifact does not collapse evidence, meaning, decisions, and execution  
- the artifact preserves canonical mediation paths  
- the artifact does not create ownership conflicts  

## 4. Target Pillar Fit
- the destination pillar is clearly identified  
- the artifact fits that pillar’s purpose and ownership model  
- promotion would not duplicate existing canonical artifacts  

## 5. Governance Approval
- governance has reviewed the artifact  
- governance has explicitly approved promotion  

---

# Disqualifiers

An artifact must not be promoted if it:

- depends on direct **Decision Intelligence System → Strategy / Governance** flow  
- bypasses the **Customer Outcomes System**  
- carries experimental ambiguity not resolved through review  
- assumes production authority prematurely  
- conflicts with approved architectural boundaries  

---

# Promotion Evidence Package

A promotion review should include:

- experiment artifact  
- experiment summary  
- documented observations  
- outcome evaluation inputs  
- boundary assessment  
- target pillar rationale  
- required cleanup items, if any  

---

# Summary

The **Lab Promotion Criteria** ensure that experimental work enters the canonical architecture only when it is:

- validated  
- bounded  
- architecturally consistent  
- formally approved
